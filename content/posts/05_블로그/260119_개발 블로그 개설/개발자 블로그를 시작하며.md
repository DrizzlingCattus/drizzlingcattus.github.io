# 배경

최근에 너무 업무에만 치우쳐서 살게되니 겉으로 안드러나는 결과물만 만들게 되어 매너리즘에 빠지게 된다. 글이든 사이드 프로젝트든 아웃풋을 만들어내고 이를 통해 외부로 소통을 하고 싶다는 욕구가 늘어가게 되어 작지만 쉽게 할 수 있는 글쓰기부터 해보자는 결심이 섰다.

너무 욕심은 내지 말고 한달에 2~4건 정도 글을 써본다는 마음가짐으로 시작해보기로한다. (꾸준히 포기하지말고 써보기! ✏️)

예전에 몇번인가 개발 블로그를 운영해보려는 시도를 한적 있는데, 매번 몇달하다가 실패했었다. 그 이유를 생각해보면 내가 완벽하게 글을 쓰려는 욕심에 의해 너무 많은 시간을 투자하다보니 제풀에 지쳐떨어졌던 기억이 난다. 그런 의미에서 이번에는 조금 가볍게 해보려 한다.

# Static Site Genator를 선택하다

최대한 빠르고 간단하게 블로그를 구축하고 싶고, markdown 형식의 파일을 만들어서 이를 기준으로 블로그를 운영하고 싶었다. 그럼 static site generator를 사용하는게 좋겠다 판단이 들어 적당히 후보군을 살펴봤다.

* `jeykell` : 풍부한 기능, 간단한 github 통합. 하지만 많은 기능으로 인한 복잡성과 무거움.
* `hugo` : 적당한 기능, 좀더 개발자 친화적. 조금 번거롭지만 정적페이지 생성 속도가 비교적 빠름.

난 어차피 많은 기능이 필요하지 않았고, 시작은 간단한 테마에 글만 작성할 수 있으면 되었기에 큰 고민없이 `hugo`를 선택했다. 필요하면 나중에 Google Analytics든 뭐든 붙이면 그만이다.

# 레이아웃 설정

Hugo의 레이아웃은 Site Configuration File을 수정하면 간단하게 변경이 가능하다.

먼저, 4개 메뉴를 순서대로 설정하고

```yaml
menu:
  main:
    - name: Home
      url: ""
      weight: 1
    - name: Categories
      url: "categories/"
      weight: 2
    - name: Tags
      url: "tags/"
      weight: 3
    - name: About
      url: "about/"
      weight: 4
```

footer를 약식으로 텍스트만 설정해준뒤

```yaml
params:
  description: "FightingYK"
  footer: "&copy; FightingYK | [Github](https://github.com/DrizzlingCattus)"
```

각 페이지의 url 패턴을 날짜 + 파일이름을 기준으로 설정해주면 끝!

```yaml
permalinks:
  post: "/post/:year/:month/:day/:contentbasename"
```

# 중요한 것

이렇게 만들어두면 뭐하겠나? 다른것보다도 중요한건 꾸준히하는게 제일 중요하다. 이를 위해 몇 가지 명심할 내용을 내 나름 마음에 새겨보며 마친다.

* 너무 대단한걸 기록하려하지말고 간단한 것이라도 기록해보자는 마음가짐으로 시작한다.
* 글이 길어질 것 같다면 시리즈로 나눠서 작성해라. 너무 길어지면 지친다.
* 내가 업무를 하면서 느꼈던 점이나 배운점들을 나의 언어로 기록하는게 중요하다.
* 기존에 있던 학습용도의 지식은 private 기록환경에 잘 쌓아두는 것으로도 충분하고, 이를 취합하여 구조화/유추/근거를 만드는 글을 작성하는게 중요하다.
